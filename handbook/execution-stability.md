# 执行稳定性协议 v1.2.2

## 解决的问题

本文件专门解决：

1. 第二页或后续页面生成卡住。
2. 图片比例从 3:4 漂移到 2:3 / A4。
3. 用户说“整理成学习笔记”时只输出方案，没有生成成品图。
4. 多页任务一次性批量生成导致超时。

## 默认行为

用户没有明确要求多页时，默认生成 1 张 3:4 总览学习笔记图。

用户没有明确说“只要方案 / 先不要生图”时，不要只停在文字方案。

## 多页模式

只有明确出现以下表达时才进入多页模式：

- 多页
- 系列图
- 拆成 N 页
- 每章一页
- 逐页生成

多页模式必须分两步：

1. 先规划 `page-outline` 和 `page-prompts`。
2. 再逐页单独生图。

禁止一次性批量生成多张图。

## 单页生图超时处理

180 秒无结果即视为超时。

### 唯一一次降载重试

重试时减少：

- 可见汉字数量，控制到 60-90 个可见汉字以内
- 要点数量
- 高亮词数量
- 图解复杂度
- 装饰元素

如果降载重试仍然超时，停止继续生成，输出失败原因和精简 prompt。可以建议改成无文字底图 + 后期叠字，但不要自动发起第二次重试。

## 3:4 固定规则

每个 final prompt 第一行必须写：

```text
STRICT 3:4 VERTICAL IMAGE ONLY. Portrait canvas, 1536x2048 composition. Do not use 2:3, A4, square, landscape, or any other ratio.
```

如果输出比例不是 3:4，优先修比例，不先修内容。

## 图片模型职责边界

图片模型只负责画当前页。

不要把这些内容发给图片模型：

- 原文全文
- 多页方案全文
- 历史版本说明
- Skill 工作流解释
- 大段中文正文
- 多种可选版式

最终 prompt 必须短、确定、单页化。


## 逐页进度报告

多页模式必须在每页完成后报告进度，例如：

```plain text
第 1/5 页已完成，准备生成第 2 页。
```

如果某页超时，必须报告：

```plain text
第 2/5 页超过 180 秒未返回，已终止本次调用，准备降载重试。
```

## 禁止同一脚本循环生成多页

每次工具调用只生成一页。

禁止在同一个脚本中循环生成多页，尤其禁止：

```plain text
for page in pages:
    generate_image(page)
```

这种批量循环会导致没有中间返回，一旦某页卡住，整批任务无法恢复。

## 降载重试只做一次

同一页超时后，只允许降载重试一次。

降载版只保留：

- 标题
- 一个简单图解
- 3 条短句
- 一句复习提醒

如果仍失败，停止继续重试，输出精简 prompt 和失败原因。

## 文字错误处理

如果出现错字、重复、乱码或排版挤压，先减少文字，而不是叠加更多纠错指令。

处理顺序：

1. 减少可见汉字数量。
2. 长句改为关键词或短标签。
3. 必要时改成无文字底图 + 后期叠字建议。
4. 再重生。

## 安装位置核验

更新 Skill 时，不能只修改工作区临时副本。

必须确认新版 ZIP 已生成，并提醒用户替换实际安装目录中的旧版 Skill 文件夹：

```plain text
~/.codex/skills/study-notes-studio/
C:\Users\<用户名>\.codex\skills\study-notes-studio\
```
