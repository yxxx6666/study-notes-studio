# 出图引擎路由协议 v1.4.1

## 目标

避免在用户明确要求某种出图方式时擅自切换。

## 出图方式优先级（v1.4.1 写死默认）

**默认且强制使用图片生成模型（image_gen / AI 生图）。** 这是写死的默认值。

- 用户没有指定出图方式时：**必须**使用图片生成模型，不得改用程序绘制或 HTML。
- 用户说“使用 image 生图模型 / image_gen / AI 生图 / 图片模型生成”：使用图片生成模型（与默认一致）。
- 用户说“禁止 HTML”：不得使用 HTML、浏览器截图或 HTML 导出流程，也不要把 HTML 作为首选建议。
- **只有**用户明确说出“固定像素 / 确定性 PNG / 程序绘制 / Pillow 画 / 禁止生图模型”时，才允许使用 Python / Pillow 直接绘制 PNG。

### 禁止借口切换

禁止以下任何理由擅自从图片模型切到程序绘制或 HTML：

- “中文要可读 / 字体更清晰”
- “要严格控制 3:4 / 像素 / 版面”
- “程序绘制更稳 / 不会漂移”

中文可读和比例问题必须通过 prompt 和比例重试在图片模型内解决，而不是换引擎。

## 禁止擅自切换

如果用户明确说“使用图片模型生成”，即使图片比例第一次失败，也不能直接切换到 Python / Pillow / HTML。

正确处理：

1. 报告比例失败。
2. 用更强的 3:4 prompt 重新调用图片模型。
3. 重试后仍失败，给出两种选择：继续图片模型重试，或改用确定性 PNG。

## HTML 禁令

当用户说“禁止 HTML”时：

- 不使用 HTML。
- 不用浏览器截图。
- 不建议“HTML 排版后导出”。
- 可选方案只能是图片模型生成或 Python / Pillow 直接绘制 PNG。

## 交付措辞

不得说“我不再依赖生图模型”，除非用户同意改用确定性绘制。

如果用户要求图片模型：

```plain text
我会继续使用图片生成模型，并在每页生成后检查比例；如果不是 3:4，先做比例重试，不切换到其他出图方式。
```
