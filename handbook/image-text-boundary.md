# 画面文字边界协议 v1.4.1

## 目标

区分“可以画进图片的内容”和“只能放在文件名 / 交付说明 / 对话里的内容”，避免内部信息污染成品图。

## 不得画进图片的内容

以下内容禁止出现在图片画面里：

- **页码**：如 `1/4`、`2/4`、`第 1 页`。页码只用于文件名（如 `note-01.png`）或交付清单。
- **内部约束与元信息**：
  - 事实边界（如“某研究证明……”的真伪约束）
  - 生成提醒 / prompt 约束 / 系统指令
  - “不要写成……”这类写作口径元话语
  - 模式名（总览 / 长文配图 / 完整笔记）
  - 审计术语（coverage / MUST / SHOULD / 覆盖矩阵）

## 研究背景的正确处理

如果原文有研究背景或事实边界需要提示：

- 不照搬约束原话。
- 最多变成一条很轻的中性边注，例如：
  - 可用：`研究启发：长期经营更重要`
  - 禁止：`不要写成：某某证明 12 个习惯`

## 页码的正确位置

- 文件名：`study-note-01.png`、`study-note-02.png`
- 交付说明：在对话里写“共 4 页，第 1 页为总览”
- 画面内：不出现页码

## 校验

生成后检查图片画面：

- 是否出现 `数字/数字` 形式页码 → 失败，重画。
- 是否出现“不要写成 / 禁止 / prompt / 模式 / 覆盖矩阵”等内部词 → 失败，重画。
