# 反信息图漂移协议 v1.4.0

## 目标

防止 Study Notes Studio 从“学霸学习笔记”漂移成信息图、公众号知识海报、PPT 总结页或 UI 卡片网格。

## 核心判断

一张合格的学习笔记图，应该像学生亲手整理的复习页。

它可以清晰、漂亮、有结构，但不能像商业信息图模板。

## 禁止方向

生成时避免：

- 信息图风格
- 公众号知识海报
- PPT 总结页
- UI 卡片网格
- 报告式数据看板
- 大量圆角模块
- 每个模块都配图标
- 顶部大标题海报化
- 数据标签铺满页面
- 研究报告式警告框
- 商业模板感

## 推荐方向

强化：

- 手写标题
- 自然圈画
- 少量高亮
- 手绘箭头
- 轻描边分区
- 充足留白
- 一页一个核心图解
- 一页一个原创学习锚点
- 像复习页，不像海报

## 图标数量规则

每页只能有：

- 1 个核心图解
- 1 个原创学习锚点图案
- 少量手绘箭头、圈线、高亮

禁止每个模块都配一个图标。

如果一页出现 5 个以上独立图标，通常已经偏离学习笔记。

## 卡片数量规则

不要把每个知识点都装进独立圆角卡片。

推荐使用：

- 手写小标题
- 自然留白
- 轻描边区域
- 手绘括号
- 一条主路径

如果页面看起来像 UI 组件库或小红书干货模板，应重生。

## 研究内容表达规则

遇到研究、数据、论文、权威背书时，不要做成研究报告页。

正确表达：

```plain text
边注：研究支持“长期经营”，不是证明每个习惯。
```

避免表达：

```plain text
大红色警告框 + 数据卡片 + 报告式结论区
```

## 失败信号

看到以下信号，判定为“信息图漂移”：

- 像 PPT
- 像公众号头图
- 像商业信息图
- 像小红书干货海报
- 大量图标和卡片
- 缺少手写笔记感
- 页面太规整、太模板化
- 更像展示给别人看的报告，而不是自己复习用的笔记

## 修复方式

1. 减少图标。
2. 减少卡片。
3. 改成手写小标题 + 自然分区。
4. 保留一个核心图解。
5. 保留一个学习锚点。
6. 把研究和数据压成边注。
7. 加强复习提醒。

## final prompt 必加句

```plain text
Keep it as a handwritten student review note, not an infographic, not a poster, not a PPT slide, not a UI card grid. Use one core diagram and one small learning anchor motif only. Avoid many icons, many rounded cards, report-like data blocks, and commercial template feeling.
```
