# Complete Notes Mode 完整笔记模式

完整笔记模式用于“完整、不遗漏、覆盖全文、全部步骤、考试复习”等需求。

核心目标：

> 用多页 3:4 学习笔记图 + 必要文字附录，完整覆盖原文 MUST 知识点。

## 触发条件

用户出现以下表达时，自动进入完整笔记模式：

- 完整
- 不遗漏
- 覆盖全文
- 覆盖全部内容
- 全部步骤
- 详细整理
- 完整笔记
- 考试复习笔记

即使用户同时说“长文配图”，完整笔记仍优先。

## 执行流程

1. 清洗原文。
2. 拆解知识点。
3. 给知识点编号。
4. 标记 MUST / SHOULD / OPTIONAL。
5. 规划页数。
6. 建立覆盖矩阵。
7. 逐页生成 prompt。
8. 每页生成后检查：
   - 3:4 比例。
   - 中文是否可读。
   - 是否覆盖本页知识点。
   - 是否因降载丢失知识点。
9. 全部生成后做 coverage audit。
10. 输出最终完整性报告。

## 页数规则

- 每页 2–3 个主要知识点。
- 信息密度高时允许 1–2 个知识点。
- 内容很短时允许 3–4 个知识点。
- 完整模式不受 5 页上限限制。
- 超过 8 页，先询问是否分批生成。

## 文字规则

- 每页 60–100 个可见汉字。
- 复杂页面可拆页，不强塞。
- 图片上不放长段落。
- 必要细节可放“文字附录”。
- 完整性以“图中文字 + 附录说明”共同计算。

## 降级规则

降级只解决执行压力，不允许制造内容缺口。

允许：

- 减装饰。
- 减背景。
- 减图解复杂度。
- 减句子长度。

禁止：

- 删除 MUST 知识点。
- 放不下却不拆页。
- 当前页降级后仍标注完整。

放不下时拆下一页。
当前页降级后必须重新标注覆盖状态。

## 完整性报告模板

```text
原文知识点：{总数}
MUST：{数量}
SHOULD：{数量}
已覆盖：{数量}
遗漏：{数量}
重复：{数量}
压缩合并：{组数}
完整性结论：通过 / 未通过
```

只有“遗漏 MUST = 0”时，才允许说完整。
