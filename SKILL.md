# Study Notes Studio 学霸学习笔记

版本：v1.4.1

## 目标

把学习内容整理成干净、漂亮、有重点、真正适合回看的 3:4 视觉学习笔记。

漂亮只是结果。
核心是：**让知识变得更容易理解、记住和复盘。**

v1.4.0 核心契约：

> **“完整”不是感觉，是审计结果。**

只有通过知识点覆盖矩阵检查的结果，才能称为“完整”。




## v1.4.0 清爽彩色手写笔记视觉基因

默认视觉不是极简黑白，也不是商业信息图。

默认应像学霸用黑笔、荧光笔和彩色圆珠笔整理出的复习页：

- 黑色手写线条做骨架
- 荧光黄标关键词
- 浅蓝做分区和边注
- 浅绿标正向结论或稳定系统
- 浅粉 / 红色标易错提醒
- 淡橙做路径和箭头

颜色必须服务学习层级，不做装饰。

每页最多 4–5 种学习笔记色。不要纯黑白，不要过度极简，也不要满页彩色模块。

final prompt 必须加入：

```text
Use a clean colorful handwritten study-note style: black pen lines as the skeleton, with restrained study-note colors such as soft yellow highlighter, pale blue section marks, light green positive highlights, soft pink/red correction marks, and muted orange arrows. Do not make it monochrome or overly minimal. Colors should look like real student note-taking tools, not a commercial infographic palette.
```

## v1.4.0 反信息图漂移规则

如果输出看起来像信息图、公众号知识海报、PPT 总结页、UI 卡片网格或报告页，判定为失败。

学习笔记图必须像学生亲手整理的复习页：

- 手写标题
- 自然圈画
- 少量高亮
- 手绘箭头
- 轻描边分区
- 充足留白
- 一个核心图解
- 一个原创学习锚点

禁止：

- 大量圆角卡片
- 每个模块都配图标
- 报告式数据标签
- 顶部海报大标题
- 商业信息图模板感

final prompt 必须加入：

```text
Keep it as a handwritten student review note, not an infographic, not a poster, not a PPT slide, not a UI card grid. Use one core diagram and one small learning anchor motif only. Avoid many icons, many rounded cards, report-like data blocks, and commercial template feeling.
```

## v1.4.0 出图与拆页补充规则

### 1. 出图引擎写死（v1.4.1 强化）

- **默认且强制使用图片生成模型（image_gen / AI 生图）。** 这是写死的默认值，不需要用户特别说明。
- 禁止以“中文可读 / 字体清晰 / 严格 3:4 / 版面可控 / 更稳”等任何理由，擅自切换到 Python / Pillow 程序绘制或 HTML。
- 只有当用户**明确说出**“用确定性 PNG / 固定像素 / 程序绘制 / Pillow 画 / 禁止生图模型”时，才允许走程序绘制；否则一律图片模型。
- 用户说“禁止 HTML”：不得使用 HTML、浏览器截图或 HTML 导出流程，也不要主动建议 HTML 方案。
- 比例失败时：先用更强的 3:4 prompt 重新调用**图片模型**重试，不得借机切换到程序绘制。

### 2. 3:4 必须生成后验证

写入 3:4 prompt 不等于比例通过。

每页生成后必须检查实际比例；如果不是 3:4，先做一次比例专用重试。

### 3. 长清单自动拆页

当内容包含 8 个以上编号步骤、习惯、原则或方法时，不要默认强塞进 1 张图。

默认进入多页学习笔记规划：

- 每页 3–4 个习惯 / 步骤。
- 先输出 page-outline。
- 再逐页生成。
- 每页完成后检查比例与覆盖。

只有用户明确说“一张图 / 单页 / 总览图”时，才压缩成 1 张总览图，并说明不承诺展开全部细节。

## v1.4.1 内容前置理解与结构化（所有模式必跑）

出图前，无论哪种模式，都必须先完成内容前置处理，不得拿到原文直接生图：

1. 识别内容类型：列表 / 分类 / 流程 / 概念 / 对比。
2. 抽取条目并计数：数清原文真实的项数（如“12 个习惯”= 12 项）。
3. 决定呈现结构：是逐项列全，还是合并成大类。
4. 正规化：去重、合并、补缺、统一层级，再进入版面。

详见 `handbook/content-preprocessing.md`。

## v1.4.1 标题—画面数量一致性校验

**标题中出现的数量，必须和画面实际呈现的项数对得上。**

- 标题说“N 个”，画面就必须呈现 N 项；做不到就拆页呈现 N 项。
- 如果出于版面把 N 项合并成 M 个大类，标题/框架必须诚实说明，例如“12 个习惯 → 4 大类”，让“M”有交代。
- 禁止：标题写“12 个”，画面只画 1–4 却不解释。
- 生成后必须核对：标题数字 = 画面可见项数（或已注明的分类映射）。

## v1.4.1 画面文字边界

以下内容**只允许**出现在文件名、交付说明或对话里，**不得画进图片**：

- 页码（如 `1/4`、`2/4`）：只用于文件名或交付清单，不进入画面。
- 内部约束：事实边界、生成提醒、禁止写法、prompt 约束、模式名、审计术语。
- “不要写成……”这类元话语不入图；如确有研究背景需提示，最多变成很轻的中性边注（如“研究启发：长期经营更重要”），不照搬约束原话。

详见 `handbook/image-text-boundary.md`。

## 模式路由表

| 用户表达 | 默认模式 | 是否承诺完整覆盖 | 是否需要覆盖矩阵 |
|---|---|---|---|
| 整理成学习笔记 | 总览笔记 | 否 | 否 |
| 做一张学习笔记图 | 总览笔记 | 否 | 否 |
| 长文配图 | 长文配图 | 否 | 建议 |
| 章节配图 | 长文配图 | 否 | 建议 |
| 给文章配图 | 长文配图 | 否 | 建议 |
| 公众号插图 | 长文配图 | 否 | 建议 |
| 每个章节一张图 | 长文配图 | 否 | 建议 |
| 完整笔记 | 完整笔记 | 是 | 必须 |
| 全文完整整理 | 完整笔记 | 是 | 必须 |
| 不要遗漏 | 完整笔记 | 是 | 必须 |
| 覆盖全部内容 | 完整笔记 | 是 | 必须 |
| 全部步骤 | 完整笔记 | 是 | 必须 |
| 详细学习笔记 | 完整笔记 | 是 | 必须 |
| 考试复习笔记 | 完整笔记 | 是 | 必须 |

优先级：**完整笔记 > 长文配图 > 总览笔记**。

如果同一句话同时出现“长文配图”和“完整 / 不遗漏 / 覆盖全文 / 全部步骤”，必须自动切换为完整笔记模式。

## 三种模式

### 1. 总览笔记模式

用途：快速理解、一张图总览、抓主线。

交付：

- 1 张严格 3:4 竖版总览学习笔记图
- 只保留主题、框架、关键词和复习提醒
- 允许高度压缩
- 不承诺覆盖全部细节

### 2. 长文配图模式

用途：给长文、公众号、课程稿做章节插图。

交付：

- 通常 3–5 张严格 3:4 章节配图
- 每张对应一个章节或主题
- 强调视觉辅助和章节主旨
- 不承诺覆盖全部知识点

最终报告必须写明：

> 本次为长文配图，不是完整笔记；已覆盖章节主旨，未承诺覆盖全部细节。

### 3. 完整笔记模式

用途：用户要求完整、不遗漏、覆盖全文、全部步骤、考试复习或详细整理。

交付：

- 页数由信息量决定
- 每页 2–3 个主要知识点
- 不受默认 5 页限制
- 超过 8 页时先询问是否分批生成
- 必须生成知识点编号清单
- 必须生成页码—知识点覆盖矩阵
- 必须进行最终覆盖审计

## 强制意图确认规则

当用户只说“长文配图”，且原文超过 1200 字或包含 8 个以上步骤时，先提醒：

> 长文配图默认是章节插图，不承诺覆盖全部细节。  
> 如果你要完整覆盖全文，应切换为完整笔记模式。

如果用户明确说以下任一表达，不再询问，直接进入完整笔记模式：

- 完整
- 不遗漏
- 覆盖全文
- 覆盖全部内容
- 全部步骤
- 详细整理
- 完整笔记
- 考试复习笔记

## 禁止错误承诺

- 长文配图模式不得说“内容完整”。
- 长文配图模式只能说“已覆盖主要章节主旨”。
- 完整笔记模式只有通过 coverage audit 后，才能说“完整”。
- 如果 coverage audit 失败，必须说“未完整”，并列出遗漏项。
- 未通过覆盖检查时，禁止输出“完整交付”“全部完成”“无遗漏”。

## 完整性契约

进入完整笔记模式后，必须先完成：

1. 清洗原文。
2. 拆解知识点。
3. 给知识点编号：K01、K02、K03……
4. 标记 MUST / SHOULD / OPTIONAL。
5. 建立页码—知识点覆盖矩阵。
6. 逐页生成。
7. 每页生成后检查覆盖状态。
8. 最终输出 coverage audit。

最终报告固定包含：

```text
原文知识点：
MUST：
SHOULD：
已覆盖：
遗漏：
重复：
压缩合并：
完整性结论：
```

如果遗漏 MUST > 0：

- 禁止称为完整。
- 必须输出“未通过完整性检查”。

## 原创学习锚点图案

每页加入 1 个 **原创学习锚点图案**。

它不是角色，不是吉祥物，不是插画主角。
它是一个小型视觉提示符，用来帮助读者记住本页知识重点。

要求：

- 每页只放 1 个。
- 面积约占画面 6%–12%。
- 必须和本页核心知识点绑定。
- 必须使用普通学习物件、路径符号、记忆符号或抽象结构符号。
- 不使用可识别角色、不使用固定 IP、不使用拟人怪物、不使用系列化角色设定。
- 不复用任何外部案例的构图、角色、动作、轮廓或视觉壳。
- 不得抢走标题、正文、图解和复习提醒的位置。

推荐图案池：

- 出勤 / 开始：小路标、课程门牌、日历角标
- 笔记 / 记录：笔记卡、小铅笔、书签
- 发言 / 讨论：小气泡、举手符号、问号灯泡
- 复习 / 记忆：记忆抽屉、循环箭头、卡片盒
- 拖延 / 时间：小闹钟、沙漏、进度条
- 沟通 / 求助：小信封、办公室门牌、对话框
- 自信 / 心态：小旗帜、指南针、上升箭头
- 社团 / 活动：小徽章、校园地图针
- 时间管理：日程格、优先级小阶梯
- 毕业 / 职业：小简历夹、实习工牌、出口路标

图片 prompt 固定写法：

```text
Learning anchor motif: add one small original content-linked hand-drawn learning symbol, about 6%-12% of the page, placed near the key section. It should reinforce the main learning idea without becoming a mascot, character, or main illustration. Use ordinary study objects, route symbols, memory symbols, or abstract structure symbols only. Avoid any recognizable character design, creature, mascot, IP-like figure, copied composition, or recurring character system.
```

## 超时降级规则

超时降级不得造成静默丢失。

允许：

- 减少装饰
- 减少背景复杂度
- 减少图解复杂度
- 减少句子长度
- 把部分知识点移到下一页或文字附录
- 缩小、简化或删除学习锚点图案

禁止：

- 直接删除 MUST 知识点
- 删除知识点后仍标记完整
- 把完整页降级成摘要页却继续计入完整覆盖
- 为了保留装饰而牺牲知识覆盖

降级后必须重新检查覆盖矩阵。

## 视觉与输出基础规则

- 默认严格 3:4 竖版。
- 整张图片画布本身就是笔记页。
- 使用浅奶白或极浅米白作为画布底色。
- 不要生成桌面背景、纸张边缘、实拍阴影、样机场景。
- 主要用留白、小标签、柔和手绘容器分区。
- 复习提醒是画布上的平面浅黄色提示色块，不要便利贴实物感。
- 每页标题清晰，分区明确，中文可读。
- 图上文字默认控制在 60–100 个可见汉字；复杂内容宁可拆页，不强塞。

## 执行入口

1. 先根据用户表达选择模式。
2. 如果触发完整笔记，先做覆盖矩阵。
3. 如果触发长文配图，先说明不承诺完整。
4. 如果触发总览笔记，默认生成 1 张 3:4 总览图。
5. 多页生成必须逐页执行，不在同一脚本内循环批量生图。
6. 每页完成后先报告进度，再继续下一页。
