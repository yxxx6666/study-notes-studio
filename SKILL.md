# Study Notes Studio 学霸学习笔记

版本：v1.2.1

## 目标

把一段学习内容，压缩成一页真正值得回看的学习笔记。

它不是把原文换成手写体。
也不是把内容装饰得很花。
更不是把整节课塞进一张图。

它要像一个会学习、会总结、审美也在线的人做出来的笔记：

- 第一眼看到主题
- 第二眼抓住重点
- 再往下能看懂结构
- 最后知道怎么复习

漂亮只是结果。
真正的核心是：**让知识变得更容易理解、记住和复盘。**

## 适合处理的内容

- 课堂讲义、老师板书、网课转写
- 讲座、播客、访谈、会议内容
- 读书摘录、论文摘要、文章段落
- 考试复习材料、知识点清单
- 用户临时丢过来的一段复杂说明

## 每页笔记的边界

一页只讲一个小主题。

通常对应：

- 一个概念
- 一个模型
- 一个流程
- 一个对比
- 一个公式
- 一个章节小节
- 一组容易混淆的知识点

如果内容太长，先切成 2-5 页。不要为了省页数牺牲可读性。

## 工作原则

### 1. 先理解，再排版

先判断这段内容到底在讲什么。
不要直接摘句子。
不要做成流水账。

你要提炼“复习抓手”：回看这页时，最能帮助用户重新理解内容的知识点。

常见复习抓手包括：

- 定义：这个东西是什么
- 区分：它和相似概念有什么不同
- 原因：为什么会这样
- 步骤：先做什么、后做什么
- 结构：它由哪些部分组成
- 规律：可以迁移使用的结论
- 误区：最容易理解错的地方
- 例子：最能解释抽象概念的场景

### 2. 做“有收获”的笔记

一页好笔记不只是好看。
它应该让用户看完有收获。

生成前先完成五件事：

1. 给这页起一个明确标题
2. 提炼 3-5 个核心要点
3. 找出 2-4 个必须高亮的关键词
4. 选择一个最适合内容结构的版式
5. 写一句复习提醒，帮助用户记住整页

### 3. 控制信息密度

宁可少而准，不要满而乱。

每页建议：

- 标题：1 个
- 核心要点：3-5 条
- 关键词：2-4 个
- 小图解：1 个
- 复习提醒：1 句
- 小装饰：少量即可

如果用户给的内容很多，先拆页。

### 4. 审美方向

整体像“学霸认真整理后的复习笔记”。

默认生成“全画布学习笔记页”：整张图片画布本身就是笔记页。不要生成摊开的笔记本、独立纸张边缘、书脊、桌面背景、实拍阴影或样机场景，除非用户明确要求展示图。

视觉稳定性要求：

- 使用浅奶白或极浅米白作为画布底色，不要纯白空白画布。
- 整张画布直接承载笔记内容，不要可见纸张外框、纸张边缘或周围背景。
- 不要投影、页面弯曲、透视、桌面、拍摄感或纸张物体感。
- 主要用留白、小标签、柔和手绘容器来分区。
- 避免明显竖线、硬切屏、正式表格感。
- 提醒块要像画布上的平面色块，完全不要投影、浮起、贴纸厚度或 3D 样机感。
- 根据知识结构变化版式，不要每页都固定成“标题 + 左右分栏 + 小图 + 便利贴”。

提醒块规则：

- “复习提醒”默认不是便利贴实物，而是画在同一画布上的浅黄色提示色块。
- 不要阴影。
- 不要纸片边缘。
- 不要胶带。
- 不要翘角。
- 不要立体厚度。
- 不要像贴在页面上的真实便利贴。

关键词：

- 清爽
- 有层次
- 手写感
- 高亮克制
- 留白舒服
- 信息组织清楚
- 看完真的能复习

不要做成：

- 讲义截图
- PPT 页
- 海报
- 儿童贴纸画
- 密密麻麻的抄书页
- 只有装饰、没有理解的假笔记

## 使用流程

### A. 用户只要整理思路时

如果用户说“先帮我整理一下 / 看看怎么做 / 先出方案”，输出笔记方案，不生成图片。

每页方案包含：

```text
第 1 页：{标题}
内容类型：定义 / 流程 / 对比 / 分类 / 公式 / 案例
建议版式：{版式名称}
核心要点：3-5 条
高亮词：2-4 个
图解：画什么关系
复习提醒：一句话
```

### B. 用户明确要生成笔记图时

如果用户说“生成 / 做出来 / 画成笔记 / 做成图片 / 给我成品”，直接生成。

生成前快速规划每页。
然后每次生成一页。
长内容最多先生成 5 页，除非用户要求更多。

### C. 用户给的是长内容时

先切分：

- 按章节切
- 按概念切
- 按问题切
- 按流程阶段切
- 按对比组切

不要把不同主题硬塞在同一页。

## 必读手册

按任务需要读取，不要把所有内容一次性塞入上下文。

- `handbook/learning-compression.md`：如何从原文提炼复习抓手。
- `handbook/page-grammar.md`：一页学习笔记应该由哪些信息层组成。
- `handbook/layout-recipes.md`：不同内容结构适合什么版式。
- `handbook/visual-style.md`：视觉风格、颜色、字体、留白、禁忌。
- `handbook/generation-protocol.md`：生成单页笔记图的提示词写法。
- `handbook/review-rubric.md`：生成后的质量检查标准。

## 输出口径

交付前说明要短。

生成后说明：

- 共生成几页
- 每页主题是什么
- 哪页最适合复习
- 如果有明显不足，说明需要重生成的点

不要讲太多设计理论。
让笔记页自己说话。
