# Study Notes Studio 学霸学习笔记

版本：v1.2.2

## 目标

把一段学习内容，压缩成一页真正值得回看的学习笔记。

它不是把原文换成手写体。
也不是把内容装饰得很花。
更不是把整节课塞进一张图。

它要像一个会学习、会总结、审美也在线的人做出来的笔记：

- 第一眼看到主题
- 第二眼抓住重点
- 再往下能看懂结构
- 最后知道怎么复习

漂亮只是结果。
真正的核心是：**让知识变得更容易理解、记住和复盘。**

## 适合处理的内容

- 课堂讲义、老师板书、网课转写
- 讲座、播客、访谈、会议内容
- 读书摘录、论文摘要、文章段落
- 考试复习材料、知识点清单
- 用户临时丢过来的一段复杂说明

## 每页笔记的边界

一页只讲一个小主题。

通常对应：

- 一个概念
- 一个模型
- 一个流程
- 一个对比
- 一个公式
- 一个章节小节
- 一组容易混淆的知识点

默认优先压缩成 1 页总览图。只有用户明确要求“多页 / 系列图 / 拆成几页 / 逐页生成”，才切成 2-5 页。不要为了省页数牺牲可读性。

## 工作原则

### 1. 先理解，再排版

先判断这段内容到底在讲什么。
不要直接摘句子。
不要做成流水账。

你要提炼“复习抓手”：回看这页时，最能帮助用户重新理解内容的知识点。

常见复习抓手包括：

- 定义：这个东西是什么
- 区分：它和相似概念有什么不同
- 原因：为什么会这样
- 步骤：先做什么、后做什么
- 结构：它由哪些部分组成
- 规律：可以迁移使用的结论
- 误区：最容易理解错的地方
- 例子：最能解释抽象概念的场景

### 2. 做“有收获”的笔记

一页好笔记不只是好看。
它应该让用户看完有收获。

生成前先完成五件事：

1. 给这页起一个明确标题
2. 提炼 3-5 个核心要点
3. 找出 2-4 个必须高亮的关键词
4. 选择一个最适合内容结构的版式
5. 写一句复习提醒，帮助用户记住整页

### 3. 控制信息密度

宁可少而准，不要满而乱。

每页建议：

- 标题：1 个
- 核心要点：3-5 条
- 关键词：2-4 个
- 小图解：1 个
- 复习提醒：1 句
- 小装饰：少量即可

如果用户给的内容很多，先拆页。

### 4. 审美方向

整体像“学霸认真整理后的复习笔记”。

默认生成“全画布学习笔记页”：整张图片画布本身就是笔记页。不要生成摊开的笔记本、独立纸张边缘、书脊、桌面背景、实拍阴影或样机场景，除非用户明确要求展示图。

视觉稳定性要求：

- 使用浅奶白或极浅米白作为画布底色，不要纯白空白画布。
- 整张画布直接承载笔记内容，不要可见纸张外框、纸张边缘或周围背景。
- 不要投影、页面弯曲、透视、桌面、拍摄感或纸张物体感。
- 主要用留白、小标签、柔和手绘容器来分区。
- 避免明显竖线、硬切屏、正式表格感。
- 提醒块要像画布上的平面色块，完全不要投影、浮起、贴纸厚度或 3D 样机感。
- 根据知识结构变化版式，不要每页都固定成“标题 + 左右分栏 + 小图 + 便利贴”。

提醒块规则：

- “复习提醒”默认不是便利贴实物，而是画在同一画布上的浅黄色提示色块。
- 不要阴影。
- 不要纸片边缘。
- 不要胶带。
- 不要翘角。
- 不要立体厚度。
- 不要像贴在页面上的真实便利贴。

关键词：

- 清爽
- 有层次
- 手写感
- 高亮克制
- 留白舒服
- 信息组织清楚
- 看完真的能复习

不要做成：

- 讲义截图
- PPT 页
- 海报
- 儿童贴纸画
- 密密麻麻的抄书页
- 只有装饰、没有理解的假笔记

## 使用流程

### A. 用户只要整理思路时

如果用户说“先帮我整理一下 / 看看怎么做 / 先出方案”，输出笔记方案，不生成图片。

每页方案包含：

```text
第 1 页：{标题}
内容类型：定义 / 流程 / 对比 / 分类 / 公式 / 案例
建议版式：{版式名称}
核心要点：3-5 条
高亮词：2-4 个
图解：画什么关系
复习提醒：一句话
```

### B. 用户明确要生成笔记图时

如果用户说“生成 / 做出来 / 画成笔记 / 做成图片 / 给我成品 / 整理成学习笔记”，默认直接生成 1 张 3:4 总览学习笔记图。

生成前快速规划单页结构。
如果用户明确要求多页，再先输出 page-outline 和 page-prompts，然后每次只生成一页。
长内容最多先生成 5 页，除非用户要求更多。

### C. 用户给的是长内容时

先切分：

- 按章节切
- 按概念切
- 按问题切
- 按流程阶段切
- 按对比组切

不要把不同主题硬塞在同一页。


## v1.2.2 稳定执行协议

本节优先级高于旧版流程。目标是解决两类问题：**第二页生成卡住**、**画面比例漂移到 2:3 / A4**。

### 1. 默认交付物

当用户说：

- “整理成学习笔记”
- “生成学习笔记”
- “做成学习笔记图”
- “把这段内容做成笔记”

默认生成 **1 张 3:4 竖版总览学习笔记图**。

不要默认只输出文字方案。
不要默认拆成多页。
不要默认批量生成多张图。

只有用户明确说以下内容时，才只输出方案：

- “先不要生图”
- “只要规划”
- “只输出笔记方案”
- “先拆页给我看”

只有用户明确说以下内容时，才进入多页模式：

- “做成多页”
- “系列图”
- “拆成 3 页 / 5 页”
- “逐页生成”
- “每个章节一页”

### 2. 多页执行规则

多页任务必须分阶段执行：

```text
内容提纯
→ page-outline
→ page-prompts
→ 第 1 页单独生图
→ 检查比例与文字量
→ 第 2 页单独生图
→ 继续逐页执行
```

禁止一次性批量生成 2 张以上图片。
禁止把全文、全部页规划和多个页面提示词一起塞给图片模型。
每次图片模型只接收 **一页的 final prompt**。

如果上一页已正常生成，下一页仍然必须重新压缩 prompt，不能复用过长上下文。

### 3. 超时降级规则

单页生图超过 180 秒没有返回，判断为超时。

第一次超时，立即降级重试：

- 图上文字减少到 60 个左右可见汉字
- 核心要点从 5 条降到 3 条
- 关键词从 4 个降到 2 个
- 去掉复杂图解
- 只保留一个简单结构示意

第二次超时，继续降级：

- 改为无文字底图 + 后期叠字建议
- 只画标题区、三个分区和一个小图解占位
- 不要求模型生成完整中文正文

第三次超时，停止继续占用进程：

- 告知用户当前页生成器卡住
- 输出可复制的精简 final prompt
- 建议稍后重试或换模型

### 4. 固定 3:4 比例规则

所有最终学习笔记图必须是 **严格 3:4 竖版**。

最终图片 prompt 必须写：

```text
Strict vertical 3:4 aspect ratio, portrait canvas, 1536x2048 composition. Do not use 2:3, A4, square, landscape, or any other ratio.
```

不要写 “A4-style”。
不要写 “2:3”。
不要写 “优先使用 3:4 / 2:3 / A4”。
如果模型输出比例不是 3:4，必须重试，并在重试 prompt 第一行强化：

```text
IMPORTANT: exact 3:4 vertical image only. The final canvas must be 3:4, not 2:3 or A4.
```

### 5. 图上文字预算

默认单页只保留：

- 1 个标题：8-12 个字
- 1 句核心判断：不超过 18 个字
- 3 条短要点：每条 8-14 个字
- 2 个高亮词
- 1 句复习提醒：不超过 18 个字

中文总量建议控制在 **60-90 个可见汉字**。
复杂解释留在文字说明里，不塞进图片。

### 6. 图片模型只负责画

不要让图片模型同时负责阅读长文、理解全文、拆页、写中文、设计版式和画图。

正确流程：

```text
文本模型提炼内容
文本模型确定页面结构
文本模型写短 final prompt
图片模型只按 final prompt 画一页
```

最终 final prompt 不包含：

- 原文全文
- 多页方案全文
- 历史版本规则
- Skill 理论说明
- 大量中文段落
- 多种可选版式

### 7. 逐页进度报告

多页模式不能长时间静默执行。

每完成一页并准备生成下一页前，必须向用户报告一次进度：

```text
第 1/5 页已完成，准备生成第 2 页。
```

如果某页超时，也要立即报告：

```text
第 2/5 页超过 180 秒未返回，已终止本次调用，准备降载重试。
```

### 8. 禁止脚本内循环批量生图

即使是逐页生成，也不能在同一个脚本或同一次工具调用中循环生成多页。

正确做法：

```text
一次工具调用 = 一页图片
一页完成 = 报告进度 = 再进入下一页
```

禁止做法：

```text
for page in pages:
    generate_image(page)
```

这种做法会导致中间无反馈，一旦第 2 页卡住，整批任务都无法恢复。

### 9. 降载重试只做一次

同一页超时后，只允许降载重试一次。

降载重试版本只保留：

- 标题
- 一个简单图解
- 3 条短句
- 一句复习提醒

如果降载重试仍然失败，不继续叠加提示词，不继续等待，不继续无限重试。

### 10. 文字错误处理

如果图中文字出现错字、重复、乱码或排版挤压，不要继续在原 prompt 上叠加“请修正错字”等纠错要求。

正确做法：

1. 先减少图上中文数量。
2. 把长句改成关键词或短标签。
3. 必要时改成无文字底图 + 后期叠字建议。
4. 再重新生成。

原因：中文错误通常来自文字负载过高，而不是缺少纠错指令。

### 11. 安装位置核验

更新 Skill 时，必须区分：

- 工作区临时副本
- 新打包 ZIP
- 用户实际安装目录中的 Skill

不能只修改工作区临时副本就认为已更新完成。

交付时必须提醒：安装新版 ZIP 后，需要替换实际安装目录中的旧版 Skill 文件夹，例如：

```plain text
~/.codex/skills/study-notes-studio/
```

如果用户在 Windows Codex 中使用，也需要替换对应的：

```plain text
C:\Users\<用户名>\.codex\skills\study-notes-studio\
```


## 必读手册

按任务需要读取，不要把所有内容一次性塞入上下文。

- `handbook/learning-compression.md`：如何从原文提炼复习抓手。
- `handbook/page-grammar.md`：一页学习笔记应该由哪些信息层组成。
- `handbook/layout-recipes.md`：不同内容结构适合什么版式。
- `handbook/visual-style.md`：视觉风格、颜色、字体、留白、禁忌。
- `handbook/generation-protocol.md`：生成单页笔记图的提示词写法。
- `handbook/review-rubric.md`：生成后的质量检查标准。

## 输出口径

交付前说明要短。

生成后说明：

- 共生成几页
- 每页主题是什么
- 哪页最适合复习
- 如果有明显不足，说明需要重生成的点

不要讲太多设计理论。
让笔记页自己说话。
